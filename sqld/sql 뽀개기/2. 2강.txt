1. 데이터베이스
  1) 정의
    - 여러 응용 시스템이 공용하기 위해 최소의 중복으로 
      통합, 저장된 운영 데이터의 집합

  2) 데이터 종류(통저운공)
    - 통합(Integrated) 데이터
        - 똑같은 데이터가 원칙적으로 중복되어 있지 않음
        - 최소 중복 or 통제된 중복

    - 저장(Stored) 데이터
        - 컴퓨터가 접근할 수 있는 저장 매체에 저장된 데이터

    - 운영(Operational) 데이터
        - 어떤 조직의 고유 기능 수행을 위한 데이터

    - 공용(Shared) 데이터
        - 조직에 있는 여러 응용 시스템들이 공동으로 생성하고 유지하며 
          이용하는 공동의 데이터

  3) 특징
    - 실시간 접근
    - 계속적인 변화 : 갱신, 삽입, 삭제 등의 연산에 의해 내용이 동적으로 변경됨
    - 동시 공유
    - 내용에 의한 참조 : 저장된 데이터의 내용을 이용해 데이터에 접근 가능

  4) 관점
    - 물리적 관점
      - 실제 저장장치에 저장된 데이터
      - 저장 레코드, 포인터, 인덱스 등
      - 비트, 바이트 등 물리적 요소로 구분

    - 논리적 관점
      - 일반 사용자의 입장에서 본 개념적 구성 양식/논리적 배치
      - 개체와 엔터티로 데이터베이스 표현하고 객체로써 서로 구분
    
2. DBMS(DataBase Management System)
  1) 정의
    - 응용 프로그램과 DataBase 사이의 중재자로서 모든 응용프로그램들이
      데이터를 공유할 수 있게 관리해주는 시스템
    - 파일 시스템에서 야기된 데이터의 종속성과 중복성 문제 해결하기 위한 
      시스템

  2) 구성요소
    - 자료정의기      : 스키마 입력하고 자료사전에 저장함
    - 질의처리기      : 쿼리문 수행
    - 트랜잭션 관리기 : 데이터베이스 프로그램을 병행 제어
    - 저장 관리기     : 데이터를 하드디스크에 저장하고 읽기
