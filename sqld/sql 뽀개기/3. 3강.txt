1. 트랜잭션
  1) 트랜잭션 정의
    - 데이터베이스에서 한 번에 수행되어야 할 연산의 집합
    - 하나의 논리적 기능을 수행하기 위한 작업의 단위

  2) 트랜잭션 상태
    - 활동(Active)
      - 초기 상태, 트랜잭션 시작하거나 실행 중
    - 부분 완료(Partial Committed)
      - 마지막 명령문 실행 후 상태, commit 직전 상태
    - 실패(Failed)
      - 정상적인 실행이 더 이상 진행될 수 없을 때 상태, 중단된 상태
    - 철회(Aborted)
      - 트랜잭션이 실행에 실패하여 취소되고 트랜잭션 시작 전 상태로 복원된 
        상태, rollback 연산 수행한 상태
    - 완료(Committed) 
      - 트랜잭션이 성공적으로 완료된 후 Commit 연산을 수행한 상태
    
  3) 트랜잭션 특성 (원일격지)
    - 원자성(Atomicity)
      - 트랜잭션에 포함된 연산은 모두 다 수행되거나 전부 실행되지 말아야 한다
    - 일관성(Consistency)
      - 트랜잭션은 데이터의 일관성을 보장해야 한다.
    - 격리성(Isolation)
      - 각 트랜잭션은 동시에 수행되지 않고 각각 고립되어 실행되어야 한다.
    - 지속성(Durability)
      - 트랜잭션이 성공적으로 완료된 후에, 데이터가 유실되지 않아야 한다.
