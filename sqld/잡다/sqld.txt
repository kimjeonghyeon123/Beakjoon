1. 데이터 모델링의 중요성 및 유의점
  1) 중복 
      - 같은 시간에 같은 데이터를 제공한다
  2) 비유연성 
      - 사소한 업무변화에 데이터 모델이 수시로 변경되면 안됨
      - 데이터 정의를 사용해 프로세스와 분리
  3) 비일관성
      - 데이터 간 상호 연관 관계에 대해 명확히 정의해야 함

2. 데이터 모델링  
  1) 개념적, 논리적, 물리적 데이터 모델링

3. 데이터 독리벙 요소
  1) 외부 스키마 : 개개인의 사용자가 보는 개인적인 DB 스키마
      - 사용자에게 보여주는 뷰
  2) 개념 스키마 : 모든 사용자 관점을 통합한 DB
      - 데이터 베이스의 전체적인 논리적 구조
      - 데이터 베이스에 실제로 어떤 데이터가 저장되었으며 데이터간의 관계는   
        어떻게 되는가
      - DB 당 하나만 존재
  3) 내부 스키마 : 물리적 저장장치 입장에서 DB가 저장되는 방법을 기술
